[HANDOFF-2A: DRAFT — Story-first]

I gave the AI a design reference.
It gave me a screen.

I gave it another prompt.
It gave me another screen.

Six screens later, the UI looked clean.
Nothing worked.

This is what actually happens when you try to build a real app with AI.

I was building a personal fitness tracker — something curated to how I train, not a clone of an existing app. Seemed like the perfect vibe-coding project. Reference design in. Let the agent build it out.

The agent delivered. Screen after screen. The layouts were there. The components rendered. It looked like an app.

But the features didn't work.
The UX simplicity was missing.
The end-to-end flows? Broken at every handoff.

Here's what I learned:

AI is excellent at building things *separately*.
It struggles to make them *work together*.

The UI exists. The backend call exists. Connecting them in a way that actually feels right to a user — that's where it falls apart.

I ended up refining it screen by screen. Flow by flow. The moment I thought one section was solid, the next integration broke something upstream.

This wasn't boilerplate. There was no template the agent could snap into place. Every transition, every edge case, every "what happens when the user does X" — that was mine to solve.

It is still not done.

Building with AI is fast *and* slow in ways I didn't expect. The raw output comes quickly. The quality comes slowly. And the "useful" part — the part where a real user can pick it up and go — that takes longer than either.

If you're experimenting with AI-assisted builds right now: what broke first for you — the UI fidelity, the feature logic, or the end-to-end flow?
