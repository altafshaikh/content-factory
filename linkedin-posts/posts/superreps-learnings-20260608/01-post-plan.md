# Post Plan — SuperReps Learnings

## Core Insight
Building a real app with AI is a screen-by-screen grind — the gap between a beautiful design reference and a working, usable product is where all the actual work lives.

## Target Audience
Software engineers, product builders, and technical founders who are actively experimenting with AI-assisted development or vibe coding — people who've tried it or are about to.

## Structure
1. The assumption: AI + a design reference = working app. Sounds simple.
2. The reality check: screens appeared, but features didn't work and UX simplicity was gone
3. The learning: refinement is screen-by-screen, flow-by-flow — not one-shot
4. The implication: "designed" ≠ "useful" — integration, transitions, and end-to-end flow are the hard parts AI still can't own

## CTA
What's your experience building real features with AI? Where did things start breaking down — the UI, the logic, or the flow?

## Tone Range
conversational / builder-honest / vulnerable

## Hashtags
#AIEngineering #VibeCoding #BuildInPublic #AITools #SoftwareEngineering

## Image Brief
Numbers/data to carry:
- "1 prompt" → hook stat (what everyone expects)
- "Screen by screen" — the reality label
- 3 failure modes: UI fidelity / feature integration / UX flow
- Tagline: "AI builds fast. Polish takes time."
- Before/after framing: "What you expect" vs "What you actually ship"

## Angle Assignments
Fixed angles (N=2):
- **Angle A: Story-first** — Open with the moment of realizing a single prompt wasn't going to be enough. Let the insight surface through the experience of building screen by screen.
- **Angle B: Bold claim-first** — Lead with the hard truth about AI-assisted building, then back it with the story from the fitness app project.
