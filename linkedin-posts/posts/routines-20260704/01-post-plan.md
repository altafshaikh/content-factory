# Post Plan — routines

**Source idea:** ROUTINES.MD
**Date:** 2026-07-04
**N:** 3

---

## Core Insight
Claude Code Routines shift AI from a tool you manually trigger to an autonomous agent with its own job description — the bottleneck moves from "did you remember to press start?" to "did you write a good enough prompt?"

## Target Audience
Mid-to-senior engineers and solo technical builders who actively use AI coding tools but still run them manually — the ones who've thought about automation but haven't set anything to run on its own

## Structure
1. Open: frame the old reality (AI as something you manually trigger = you are the bottleneck)
2. The shift: what Routines change — the AI gets a schedule, a brief, a definition of done
3. The real unlock: when you close the laptop, work keeps happening
4. The mental model shift: you're no longer the operator, you're the employer

## CTA
If you're using AI tools manually today, what's the first task you'd set to auto-run?

## Tone Range
Conversational with a builder-to-builder edge — honest, specific, not breathless

## Hashtags
