## Core Insight
Claude Code Routines shifts AI from a manually triggered tool into a persistent cloud agent that works on a schedule — even when your laptop is closed.

## Target Audience
Entry-to-mid career software engineers in Indian tech hubs (Mumbai, Bengaluru, Delhi) who use AI coding tools and want to stop being the bottleneck in their own workflows.

## Structure
1. The old constraint: AI automation dies the moment you close your laptop or terminal
2. What Claude Code Routines actually does: define a task + schedule, Claude runs it in the cloud autonomously
3. What it unlocks for a solo dev or small team: PR reviews, issue triage, deploys, standup prep — all running without you
4. The deeper shift: from scripts that follow rules and break on edge cases, to agents that reason through them

## CTA
What would you automate first if your AI worked around the clock?

## Tone Range
Provocative + educational

## Performance Context
- Best performing angle so far: Bold claim-first on developer tool topics — claude-vs-codex hit 5.09% engagement rate (above account baseline of 3.77%)
- Hook styles that drove comments: comparison framing (A vs B) — audiences have a stake in the outcome
- Hook styles that drove passive reactions only: insufficient data as yet
- Topics/themes that resonated: developer tool comparisons, AI productivity, practical engineering content
- What to avoid repeating: abstract strategy or management framing; audience is majority early-career engineers, not managers
- Implication for THIS post: the "Scripts vs Agents" contrast should be central — it's exactly the kind of A-vs-B comparison that drove the top engagement. Lead with the provocative challenge to a familiar tool (GitHub Actions), not with feature announcement framing.

## Angle Assignments
- Angle A: Story-first — open with the "laptop closed, automation still running" scene; let the shift emerge from lived contrast
- Angle B: Bold claim-first — lead with the direct challenge to GitHub Actions / scripts, then build the case
- Angle C: Tactical/how-to-first — lead with the 4-step setup (Decide, Create, Prompt, Connect), walk through each concretely

Chose Tactical for Angle C because decision rule 2 matched first: the raw idea explicitly describes a concrete repeatable setup process (Decide → Create → Prompt → Connect) and includes an anatomy of a perfect prompt (Role, Task, Steps, Output, Constraints, Error Handling). This produces content Angle A and B structurally cannot — a practical how-to that readers can act on immediately.
